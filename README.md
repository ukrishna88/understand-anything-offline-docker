# Understand Anything — Offline Docker Dashboard

A sandboxed, network-isolated Docker dashboard for visualizing codebase knowledge graphs. One image works for any project.

**This is a local development tool. Not for cloud deployment.**

## What is this?

The [understand-anything](https://github.com/nicholasgriffintn/understand-anything) plugin for Claude Code analyzes your codebase and produces a `knowledge-graph.json` — a map of all files, functions, classes, and their relationships. This repo provides a **pre-built Docker image** to view that graph in an interactive dashboard, with zero internet access at runtime.

## Prerequisites

- **Docker Desktop** — [Download here](https://www.docker.com/products/docker-desktop/)
- A project with a `knowledge-graph.json` (generated by running `/understand` in Claude Code, or pulled from git if a teammate generated it)

## Installation

### Step 1: Download

Go to the [Releases](https://github.com/ukrishna88/understand-anything-offline-docker/releases) page and download:
- `understand-anything-dashboard.tar.gz` (~193MB — the Docker image)

### Step 2: Run the installer

```bash
# Clone or download this repo
git clone https://github.com/ukrishna88/understand-anything-offline-docker.git
cd understand-anything-offline-docker

# Place the downloaded .tar.gz in this directory, then:
chmod +x install.sh
./install.sh
```

The installer will:
1. Check Docker is running
2. Load the Docker image (no internet needed)
3. Install a `understand-dashboard` command
4. Optionally install the Claude Code skill (for generating graphs)

### Step 3: View any project

```bash
understand-dashboard /path/to/your/project
```

Open the URL shown in the terminal. Done.

## Usage

```bash
# Start the dashboard for a project
understand-dashboard ~/Desktop/my-project

# Switch to a different project
understand-dashboard ~/Desktop/other-project

# Stop
cd ~/.understand-anything-docker && docker compose down

# Check logs / get the URL again
docker logs understand-anything
```

## How the knowledge graph gets generated

The Docker dashboard is **view-only**. To create or update a `knowledge-graph.json`:

1. Open Claude Code in your project directory
2. Run `/understand`
3. Commit the generated `.understand-anything/knowledge-graph.json` to git

Only one team member needs to do this. Everyone else pulls and views.

## Team Workflow

```
Developer A (has Claude Code + understand-anything skill):
  1. Runs /understand → generates knowledge-graph.json
  2. Commits and pushes to git

Developer B, C, D (view-only):
  1. git pull
  2. understand-dashboard .
  3. Opens the dashboard URL in browser
  4. Explores architecture, layers, guided tour
```

## Security

| Property | Detail |
|----------|--------|
| Internet access | **None** — DNS set to 0.0.0.0, cannot resolve any hostname |
| Source code access | **Read-only** — mounted as `ro`, container cannot modify files |
| Writable scope | Only `.understand-anything/` directory |
| Secrets | **None** — no API keys, credentials, or env vars required |
| Dashboard auth | Random token generated per launch (shown in `docker logs`) |
| Network exposure | `localhost` only — not accessible from other machines |

## Verify sandbox isolation

```bash
docker exec understand-anything node -e \
  "fetch('https://example.com',{signal:AbortSignal.timeout(3000)}).then(()=>console.log('HAS INTERNET')).catch(()=>console.log('SANDBOXED'))"
# Should print: SANDBOXED
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| "Docker is not installed" | Install Docker Desktop and restart |
| "Docker is not running" | Start Docker Desktop |
| "No knowledge-graph.json found" | Run `/understand` in Claude Code, or `git pull` if a teammate generated it |
| Port 5173 in use | `PORT=5174 understand-dashboard /path/to/repo` |
| Dashboard shows stale data | Refresh the browser — it re-reads the JSON on load |
| Want to update the image | Download the latest release and re-run `install.sh` |

## Uninstall

```bash
# Remove the container and image
docker compose -f ~/.understand-anything-docker/docker-compose.yml down
docker rmi understand-anything-dashboard:latest

# Remove the compose directory
rm -rf ~/.understand-anything-docker

# Remove the command symlink
rm -f /usr/local/bin/understand-dashboard
rm -f ~/.local/bin/understand-dashboard
```
